MVP Feature Checklist

(Internal Execution Document)

Core SDK (Must Have)
  •  MCP enforcement layer
  •  ControlledAgent abstraction
  •  Deterministic simulation engine
  •  Versioned dataset handling
  •  JSON schema validation
  •  SHA-256 audit logging
  •  Sanitized logging (no secrets)

⸻

Explicit Exclusions (Must Never Exist)
  •  Broker integrations
  •  Real execution paths
  •  Advisory language generation
  •  Performance guarantees
  •  User financial data storage

⸻

Stability & Quality
  •  Unit tests for MCP rules
  •  Simulation reproducibility tests
  •  Expanded integration tests
  •  CI hardening

⸻

Documentation
  •  API Specification
  •  Neutral README
  •  Minimal examples expansion
  •  Contribution guidelines

⸻

Out of Scope (Intentionally Deferred)
  •  UI / frontend
  •  Cloud execution
  •  Subscription logic
  •  Ranking systems
  •  Notifications

⸻

Definition of MVP Completion

The MVP is complete when:
  •  The SDK can safely generate, simulate, and audit portfolio configurations
  •  All outputs are reproducible
  •  No execution or advisory risk exists
  •  The repository can be publicly shared without ambiguity

⸻

Final Note

These documents intentionally prioritize clarity, neutrality, and safety over persuasion or marketing.
They are designed to remain valid as long-term public records.
